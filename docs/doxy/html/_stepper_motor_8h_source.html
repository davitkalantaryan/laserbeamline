26 // - There are 3 pure virtual functions which have to be implemented:
27 // GoSteps(int), Stop(), GetStatus().
28 // - The default version of Reset() does nothing, thus it usually has to be
29 // overridden too.
30 // - The default version of GoToInitiator() does nothing. It should be overridden
31 // if an initiator (e. g. an end position switch) is provided by the hardware.
32 // - It is highly recommended to override GetVersion().
33 // - The default versions of SetParameter() and GetParameter() do nothing. They can
34 // be overridden for "character oriented" motor controllers, to provide an interface
35 // for an interactiv command interpreter (in a test phase, e. g.). Avoid to call these
36 // functions in an "ordinary" application (calls will hardly work with different
37 // controllers).
38 // - All other virtual functions should be re-implemented for a given piece of
39 // hardware, if that hardware provides a better or more efficient way to perform
40 // the function.
